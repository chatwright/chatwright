# Chatwright specifications

The specification separates product intent from implementation evidence:

- [`ideas/`](ideas/README.md) explains why Chatwright should exist and the
  smallest valuable product promise.
- [`features/`](features/README.md) defines user-visible capability boundaries
  and acceptance criteria.
- [`decisions/`](decisions/README.md) records current direction without freezing
  unresolved APIs.
- [`research/`](research/README.md) holds the investigation backlog that must be
  answered with evidence.
- [`plans/`](plans/README.md) turns approved scope into sequenced delivery.

Narrative sequencing and long-term direction live in the
[`roadmap`](../docs/roadmap.md), [product strategy](../docs/product-strategy.md)
and [Cloud strategy](../docs/cloud-strategy.md).

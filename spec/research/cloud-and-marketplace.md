# Research: Cloud and Marketplace strategy

**Date:** 2026-07-22
**Owner:** alex
**Status:** Proposed
**Consumed by:** [`cloud`](../features/chatwright/cloud/README.md), [`marketplace`](../features/chatwright/marketplace/README.md)

## Purpose

Validate the smallest useful Cloud service and ecosystem seams without expanding
the local MVP or prematurely choosing prices, quotas or marketplace machinery.

## Investigation backlog

| ID | Question | Required evidence and output |
|---|---|---|
| I-55 | Which additive Cloud job creates the strongest voluntary sign-in after repeat local use? | Interviews and clickable tests comparing sync, hosted reports, web Studio and bounded Cloud Run. |
| I-56 | Which illustrative free-tier capabilities and limits encourage adoption at sustainable cost? | Cost model and staged quota experiment; no pricing decision required. |
| I-57 | Which sync/export identity model preserves offline ownership and survives account disconnection? | Local-first conflict/recovery prototype with project, scenario, run and asset revisions. |
| I-58 | Which report schema supports regression, quality, latency, AI cost, model, coverage, flake, platform and UX trends? | Report contract linking every aggregate to versioned evidence and judgement metadata. |
| I-59 | What isolation, secret, retention and redaction model is required for hosted execution? | Threat model and two-tenant execution spike with explicit data lifecycle. |
| I-60 | Which managed AI evaluation job reaches an acceptable evidence and agreement threshold? | Blind evaluation set, agreement/cost study and review policy. |
| I-61 | How can model and prompt benchmarks remain comparable across provider/model change? | Versioned benchmark manifest, repeated-run variance analysis and deprecation policy. |
| I-62 | Which sampling, stopping, clustering and outlier rules make swarm reports useful? | Campaign simulation showing budget, coverage proxy, duplicate consolidation and preservation of rare severe failures. |
| I-63 | How are AI-discovered failures reviewed and exported as deterministic regressions? | Approval workflow with portable scenario diff and offline replay. |
| I-64 | What is the smallest Marketplace package/metadata format for personas, scenarios, assertions, evaluators, adapters and fixtures? | Format draft tested with one data-only and one executable asset. |
| I-65 | How should community libraries express licence, provenance, trust, compatibility, maintainers and deprecation? | Curated source-repository index prototype and supply-chain review. |
